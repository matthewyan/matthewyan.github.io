概念

  • e2e：End to End，即端到端测试。它模仿用户，从某个入口开始，逐步执行操作，直到完成某项工作。它是检查整个系统与产品行为是正常的。
  • UI测试：保证UI正常工作，一般UI测试与e2e测试会有一些重叠。

e2e与UI测试区别可以参见：What's the difference between a UI test and an E2E Test?

测试方式

回归测试

是指对软件的新版本测试时，重复执行之前某一个重要版本的所有测试用例

目的：

  • 验证之前版本产生的所有缺陷已全部被修复；
  • 确认修复这些缺陷没有引发新的缺陷

自动化测试

WebDriver

WebDriver是什么呢？即是一个工具，也是一种协议。

使用场景上，主要是用来作为自动化测试使用的。

作为工具

作为工具时，是 selenium工具集中的一个，以支持浏览器自动化。具体安装可以参考：在Mac系统下搭建Selenium环境并驱动Chrome浏览器

可以通过如下类似的脚本来控制浏览器：

from selenium import webdriver

driver = webdriver.Chrome()
driver.get('http://www.baidu.com/')

WebDriver实现原理介绍可参考：关于Selenium Webdriver 实现原理的一点思考和分享

作为协议

WebDriver是W3C推荐的浏览器交互协议，协议详情参考：WebDriver

WebDriverAgent

要使用Appium，要先理解 WebDriverAgent。它是在iOS端实现了WebDriver协议的工具，由Facebook开发，但目前由appium团队维护。

其工作原理可参考：WebDriverAgent入门篇-安装和使用 中的原理部分。

其它资料：理解和使用 WebDriverAgent

模拟器上使用步骤
  • git clone https://github.com/appium/WebDriverAgent.git
  • brew install Carthage
  • ./Script/build.sh
  • 打开 WebDriverAgent.xcodeproj
  • 选择 WebDriverAgentRunner的target，Cmd + U运行测试用例
  • 在控制台中可以看到类似：ServerURLHere->http://192.168.0.102:8100<-ServerURLHere 这样的字眼
  • 浏览器中输入：http://192.168.0.102:8100/status，即可获取当前设备状态。

其它的操作命令可以在这儿查看：Queries

除了在shell中使用curl来控制手机操作，还有一些三方封装，如：facebook-wda，其可以使用类似 driver.home() 这样简单命令进行更简单的控制。

WebDriverAgent的实现入口代码，可以在WebDriverAgentRunner的UITestingUITests.m中看到。即 testRunner接口的实现，当执行Cmd + U时，即启动了一个http server，通过接收电脑的指令以在手机端实现一些操作。
