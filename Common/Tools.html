开发知识库

常用工具

brew

在M1电脑上使用Brew，请参考：How to install Homebrew in macOS Catalina or M1 Mac

命令作用
brew unlink lib解除某个库的引用链接
brew link lib建立某个库的引用链接

统计代码行数

安装：

gem install git_stats

在工程目录执行：

git_stats generate

其它的类似cloc也可以，参考：有哪些比较好用的代码量统计工具？

生成文本类型的目录结构

  • brew install tree
  • 使用：tree -I node_modules > tree.text (表示忽略node_modules目录)

VSCode

VSCoce远程开发配置：https://juejin.im/post/5cf9d23ce51d45773d4685d9

远程开发配置好后，连接到远程服务器后，就用“打开文件夹”功能，就可以打开远端目录

功能快捷键
回退到上个位置Ctrl + -
快速打开文件Ctrl + p
代码格式化Shift + Option + F
全局替换代码Cmd + Shift + H

Gem

命令作用
gem sources查看所有的gem源
gem sources -r url删除指定的url源 (url要以/结尾)
gem sources -a url添加源
gem install lib -v "~> 1.1"安装指定版本的库

Bundler

能够跟踪并安装所需的特定版本的 gem

安装bundler：gem install bundler

安装依赖：bundle install

cocoapods

安装：sudo gem install cocoapods

如果提示路径没有权限的错误，可以用-n参数指定路径：sudo gem install cocoapods -n /usr/local/bin

命令作用
sudo gem install cocoapods安装及更新
pod setup安装spec仓库
pod repo update更新spec仓库
pod repo add [name] [url]添加三方源（会在~/.cocoapods/repos下面新增源）
Pod repo remove [name]移除相应三方源

彻底删除cocoapod:

for i in $( gem list --local --no-version | grep cocoapods );
do 
    gem uninstall $i; 
done

rm -rf ~/.cocoapods/

Fastlane

Fastlane介绍及快速集成

实现iOS和Android的发布流程的自动化，以节省发布时间。功能有如下：

  • 生成并下载开发者证书
  • iTunes Connect上创建应用
  • 打包应用
  • 实现应用截屏
  • 发布到Testflight
  • 将包、截屏等传到Appstore

Fastlane实际上是一个Ruby脚本的集合,里面的每一个工具实际就是一个Ruby脚本,用于执行特定的工作任务。

Gitlab-runner

gitlab-runner作用

我们希望提交的代码可以执行某些任务，这些任务默认是放在工程根目录下的.gitlab-ci.yml中的。而如何执行这些命令，就是通过gitlab-runner这个工具来运行的。

gitlab-runner使用

  • 安装：brew install gitlab-runner

  • 启动：brew services start gitlab-runner

  • 注册：gitlab-runner register

  • 从相应Git仓库下可以找到URL和Token，在Gitlab的settings->CI->Runners->Expand下面可以查看

  • 然后输入描述、tag(这个是比较重要的，标识)等信息

  • 设置执行环境，设置为Shell即可
