Java

能力

概览

  • 支持枚举
  • 支持泛型
  • 支持反射

注解

注解是在代码中添加元数据的机制。添加的元数据，可以在运行时由其它代码，或者在编译时由其它工具所获取的数据，并且可以根据元数据做相应处理。其在使用场景上有：

  • 编译时检查
  • 框架和库配置 （Retrofit使用注解来定义网络请求和处理响应的方式）
  • 生成代码
  • 运行时反射
  • 文档生成
  • 标记和元数据
  • 代码分析和工具集成

Java默认就提供了一些注解，如:

  • @Override: 提供了编译器级别的检查，用于确保方法的正确覆盖。
  • @Deprecated
  • @SuppressWarnings: 关掉编译器某种警告。（如@SuppressWarnings(unused)）
  • @FunctionalInterface
  • ……

自定义注解

注解是支持自定义的，如下就是一个示例：

// 注解定义 (在interface前面有@符号)
public @interface MyAnnotation {
    String value() default "";
}

// 注解使用
@MyAnnotation(value = "Hello")
public class MyClass {
    // 类的内容
}

MyClass obj = new MyClass();
Class<?> clazz = obj.getClass();
MyAnnotation annotation = clazz.getAnnotation(MyAnnotation.class);
String value = annotation.value();  // 注解使用

反射

反射允许程序在运行时检查、访问和修改类、接口、方法、构造函数和字段等的内部结构和状态。Java中相关反射API都位于 java.lang.reflect 包中。

Class

反射的核心，表示 Java 类的运行时信息。可以获取类的构造函数、方法、字段、注解等信息，以及创建类的实例。

Constructor

表示类的构造函数。可以通过其获取构造函数的参数类型、修饰符等信息，并用于创建类的实例。

Method

表示类方法。可以获取方法的参数类型、返回类型、修饰符等信息，并用于调用方法。

Field

表示类的字段（成员变量）。可以获取字段的类型、修饰符等信息，并用于读取或修改字段的值。

import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.Method;

class MyClass {
    private String name;

    public MyClass(String name) {
        this.name = name;
    }

    public void sayHello() {
        System.out.println("Hello, " + name + "!");
    }
}

public class Main {
    public static void main(String[] args) throws Exception {
        // 获取 MyClass 类的 Class 对象
        Class<?> clazz = MyClass.class;

        // 创建 MyClass 对象
        Constructor<?> constructor = clazz.getConstructor(String.class);
        Object myObject = constructor.newInstance("John");

        // 调用 sayHello 方法
        Method method = clazz.getMethod("sayHello");
        method.invoke(myObject);

        // 访问 name 字段
        Field field = clazz.getDeclaredField("name");
        field.setAccessible(true);
        String nameValue = (String) field.get(myObject);
        System.out.println("Name: " + nameValue);
    }
}

Modifier

提供了用于解析和操作修饰符的方法。可以检查字段、方法、构造函数的修饰符信息，例如是否是 public、private、static 等。

Proxy

创建动态代理对象的工具类。可以在运行时创建一个实现指定接口的代理类。一般配合InvocationHandler使用，示例见InvocationHandler中。

InvocationHandler

可以拦截代理对象的方法调用并执行自定义的逻辑。 一般是配合Proxy一起使用。

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

class MyInvocationHandler implements InvocationHandler {
    private Object target;

    public MyInvocationHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        // 在方法调用前执行一些操作
        System.out.println("Before method: " + method.getName());

        // 调用目标对象的方法
        Object result = method.invoke(target, args);

        // 在方法调用后执行一些操作
        System.out.println("After method: " + method.getName());

        // 返回方法调用结果
        return result;
    }
}

interface MyHello {
    public void sayHi();
}

class MyHelloImpl implements MyHello {
    public void sayHi() {
        System.out.println("Hi");
    }
}

public class Main {
    public static void main(String[] args) {
        MyHello hello = new MyHelloImpl();  // 接口与实现关联

        // 代理的MyHello，需要是interface
        MyHello proxy = (MyHello) Proxy.newProxyInstance(
                MyHello.class.getClassLoader(),
                new Class<?>[]{MyHello.class},
                new MyInvocationHandler(hello)
        );

        proxy.sayHi();
    }
}

类/方法

内部匿名类

// 要提前定义接口，表示创建的对象的类型
interface MyInterface {
    void doSomething();
}

public class Main {
    public static void main(String[] args) {
        // 直接创建接口实例，即内部匿名类
        MyInterface myInterface = new MyInterface() {
            public void doSomething() {
                System.out.println("Doing something...");
            }
        };
        myInterface.doSomething();
    }
}

函数式接口

只包含一个抽象方法的接口，就是函数式接口。其被视为一个函数的签名，可以用在Lambda表达式中。

可以用@FunctionalInterface来显示指定接口是函数式接口

@FunctionalInterface    // 显示指定函数式接口
interface MyInterface {
    void doSomething();

    // void doOther();  // 如果指定了@FunctionalInterface，定义两个抽象接口就会失败

    default void doOther() { ... }  // 如果指定默认实现，就可以定义多个接口
}

Lambda表达式

从Java 8开始引入的Lambda表达式。其可以用来代替单个抽象方法的接口(函数式接口)。

// 也被称为 函数式接口, 被视为一个函数的签名
interface MyInterface {
    void doSomething();
}

public class Main {
    public static void main(String[] args) {
        MyInterface myInterface = () -> {
            System.out.println("Doing something...");
        };
        // 调用时仍然要指定调用的是doSomething()
        myInterface.doSomething();
    }
}

泛型

Java中的泛型，可以使用在类、接口、方法、参数等场景

// 泛型类
public class Box<T> {
    private T item;

    public void setItem(T item) {
        this.item = item;
    }

    public T getItem() {
        return item;
    }
}

// 使用泛型类
Box<Integer> integerBox = new Box<>();
integerBox.setItem(10);

// 泛型方法
public <T> void printArray(T[] array) {
    for (T item : array) {
        System.out.println(item);
    }
}

// 使用泛型方法
Integer[] intArray = { 1, 2, 3, 4, 5 };
printArray(intArray);   // 支持自动推断

通配符泛型

用于处理类型不确定的情况。用在泛型类、泛型方法和泛型接口中。(它并不是泛型定义场景使用，而是在泛型使用的场景使用)

// 表示processList可以接收各种类型的list数据 (无界通配符)
public void processList(List<?> list) {
    for (Object item : list) {
        System.out.println(item);
    }
}

// 表示接收的数据，必须是Number及其子类类型 (上界通配符)
public void processList2(List<? extends Number> list) {
    for (Object item : list) {
        System.out.println(item);
    }
}

// 使用通配符泛型
List<Integer> intList = Arrays.asList(1, 2, 3);
processList(intList);
processList2(intList);

List<String> stringList = Arrays.asList("Hello", "World");
processList(stringList);
// processList2(stringList);    // 这行代码会出错，因为参数是String列表

类型

基本类型 / 引用类型

Java中的基本类型就是int、boolean、char、byte、short、long、float 和 double。 除基本类型之外的，都是引用类型。

引用类型是通过new创建的对象，并且变量存储的实际上是对象地址。

包装类型

包装类型（Wrapper Types）是 Java 中用于包装基本类型的类。包装类型是引用类型，提供了一些额外的方法和属性，以方便开发。

其对应关系如下：

  • boolean：Boolean
  • byte：Byte
  • short：Short
  • int：Integer
  • long：Long
  • float：Float
  • double：Double
  • char：Character
自动装箱 / 自动拆箱
  • 自动装箱: 将基本类型自动转换为对应的包装类型。
  • 自动拆箱: 将包装类型自动转换为对应的基本类型。
int num = 10; // 基本类型
Integer wrappedNum = num; // 自动装箱，将 int 转换为 Integer

Integer wrappedNum1 = 20; // 包装类型
int num1 = wrappedNum1; // 自动拆箱，将 Integer 转换为 int
